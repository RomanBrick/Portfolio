Clinical ML

Applied machine learning and data engineering for clinical decision support.

I am building an applied clinical machine learning project focused on risk stratification and decision support, not automated diagnosis. The goal is to design a system that is technically sound, interpretable, and aligned with real healthcare constraints.

Why this project

I come from a clinical background (former oncologist) and a strong engineering background (DevOps / Platform engineering). Recently, I completed an MIT Data Engineering certification and am preparing to start an MBA.

This project sits at the intersection of these areas and serves as a practical way to connect clinical domain knowledge, data engineering, machine learning, and decision-making trade-offs.

What is being built

  • Admission-level feature tables from structured clinical data
  • End-to-end data preprocessing and modeling pipeline
  • Baseline clinical ML model with strict leakage prevention
  • Interpretability-first evaluation (no black-box optimization)

High-level architecture

The system is designed as a modular pipeline where the data source can be replaced (open datasets → restricted ICU datasets) without changing downstream logic.

Raw Clinical Data
↓
Data Cleaning & Validation
↓
Admission-level Feature Table
↓
Preprocessing Pipeline
↓
Baseline ML Model
↓
Risk Scores & Interpretation

Current status

Baseline completed. The current model demonstrates that using aggregated tabular data and a simple, interpretable approach, it is possible to identify a meaningful portion of high-risk admissions, at the cost of increased false alerts.
